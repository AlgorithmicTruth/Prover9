=============================================
Command-Line Reference
=============================================

LADR-2026-3C (March 2026)


This file documents command-line options for Prover9, Mace4, and
prooftrans. All other programs accept "help" as an argument for
a synopsis.


=============================================
Prover9
=============================================

Usage:

  prover9 [options] [-f file ...]
  prover9 [options] problem.p

Options:

  -h              Print help message and exit.

  -t <seconds>    Time limit. Overrides max_seconds in the input
                  file. Maps to: assign(max_seconds, <n>).

  -f <files>      Read input from one or more files instead of stdin.
                  Multiple files are concatenated. In LADR mode,
                  files contain formulas, options, and lists. In
                  TPTP mode, files contain FOF/CNF formulas and
                  include directives.

  -x              Enhanced auto mode. Maps to: set(auto2).

  -p              Fully parenthesize all output terms. Useful for
                  checking and debugging parse declarations.

  -r <dir>        Resume from checkpoint directory. Experimental.
                  See Checkpoint-Resume.txt.

  -m              Enable memory logging. Logs allocation/free
                  activity to stderr. Diagnostic use only.

TPTP flags:

  -tptp           Force TPTP input mode (read TPTP from stdin).

  -tptp_out       Force TPTP/TSTP output mode. Produces SZS status
  -tptp-out       lines and TSTP-format proofs. Normally auto-enabled
                  when TPTP input is detected.

  -ladr_out       Force native LADR output on TPTP input. Produces
  -ladr-out       standard Prover9 output with full justifications.
                  Required for prooftrans (ivy, hints, etc.).

  -comp <n>       Competition mode. Shorthand for -cores <cpus>
                  -fastPE -t <n>. Enables sliding-window scheduling
                  on all available CPUs with fast predicate
                  elimination and a time limit of <n> seconds.

  -fastPE         Enable fast predicate elimination. Imposes a
                  5-second wall-clock timeout on the predicate
                  elimination preprocessing phase. Without this
                  flag, predicate elimination runs to completion
                  (which can hang on large inputs). Can also be
                  enabled with: set(fast_pred_elim).

Strategy flags:

  -cores <n>      Sliding-window scheduler with N concurrent children
                  (1-64). Auto-enables strategy scheduling. Each child
                  runs a different ML strategy from the portfolio.
                  See Strategy-Scheduling.txt.

  -strategy <n>   Force single strategy index N from the ML portfolio.
                  Runs a deterministic single-core search with that
                  strategy's parameters. Overrides -cores.

  -strategies <n> Override Phase 1 breadth cap. Limits how many
                  strategies are tried in Phase 1 of the sliding-window
                  scheduler. See Strategy-Scheduling.txt for details.

SInE flags:

  -nosine         Disable SInE premise selection. Equivalent to:
  -no-sine        assign(sine, 0). Useful when SInE auto-enables but
  -no_sine        you want the full axiom set.

Memory flags:

  -nomem          Disable memory limit enforcement. Sets max_megs
                  to 0 (unlimited). Useful for large problems where
                  the default 48 GB limit is insufficient.

Positional arguments:

  problem.p       Read TPTP input from file. Auto-detected by .p
                  extension. Equivalent to: prover9 -f problem.p

TPTP auto-detection:

  TPTP mode is auto-enabled when any of:
    - A .p file is given (via -f or as positional argument).
    - The -tptp flag is present.
  When TPTP mode is active, TSTP output is auto-enabled unless
  -ladr_out is specified. Without -comp or -cores, TPTP mode
  runs a single deterministic search (no forking).


Examples:

  prover9 -t 60 -f problem.p           TPTP input, 60s timeout.
  prover9 -t 60 -f input.in            LADR input, 60s timeout.
  prover9 problem.p                    TPTP input, bare positional.
  prover9 -ladr_out -t 60 -f prob.p    TPTP input, LADR output.
  prover9 -tptp < input.txt            TPTP from stdin.
  prover9 < input.in                   LADR from stdin.
  prover9 -x -t 300 -f hard.p          Auto2 mode, 5 min timeout.
  prover9 -r ckpt_dir -f input.in      Resume from checkpoint.
  prover9 -comp 60 problem.p           Competition mode, 60s.
  prover9 -cores 4 -t 300 problem.p   4-core sliding window, 5 min.


=============================================
Mace4
=============================================

Usage:

  mace4 [options] [-f file]
  mace4 [options] problem.p

Basic options:

  -f <file>       Read input from file instead of stdin.
  -n <n>          Start domain size. Maps to: assign(domain_size, <n>)
                  and assign(start_size, <n>).
  -N <n>          End domain size. Maps to: assign(end_size, <n>).
  -i <n>          Domain size increment. Maps to: assign(increment, <n>).
  -t <n>          Time limit in seconds (overall).
                  Maps to: assign(max_seconds, <n>).
  -s <n>          Time limit per domain size.
                  Maps to: assign(max_seconds_per, <n>).
  -m <n>          Maximum number of models to find.
                  Maps to: assign(max_models, <n>).
  -b <n>          Memory limit in megabytes.
                  Maps to: assign(max_megs, <n>).

Model output:

  -P <0|1>        Print models in standard format.
                  Maps to: set/clear(print_models).
  -p <0|1>        Print models in tabular format.
                  Maps to: set/clear(print_models_tabular).

Search tuning:

  -O <n>          Selection order (0, 1, or 2).
                  Maps to: assign(selection_order, <n>).
  -M <n>          Selection measure (0-4).
                  Maps to: assign(selection_measure, <n>).
  -L <0|1>        Least Number Heuristic.
                  Maps to: set/clear(lnh).
  -G <0|1>        Negation propagation.
                  Maps to: set/clear(negprop).
  -H <0|1>        Negative assignment.
                  Maps to: set/clear(neg_assign).
  -I <0|1>        Negative assignment (near).
                  Maps to: set/clear(neg_assign_near).
  -J <0|1>        Negative elimination.
                  Maps to: set/clear(neg_elim).
  -K <0|1>        Negative elimination (near).
                  Maps to: set/clear(neg_elim_near).
  -R <0|1>        Integer ring arithmetic.
                  Maps to: set/clear(integer_ring).
  -S <0|1>        Skolems last.
                  Maps to: set/clear(skolems_last).
  -q <0|1>        Iterate over prime domain sizes only.
                  Maps to: set/clear(iterate_primes).
  -Q <0|1>        Iterate over non-prime domain sizes only.
                  Maps to: set/clear(iterate_nonprimes).

Diagnostic:

  -v <0|1>        Verbose output.
                  Maps to: set/clear(verbose).
  -T <0|1>        Trace search (very verbose).
                  Maps to: set/clear(trace).
  -A              Enable memory logging to stderr.

Checkpoint:

  -r <dir>        Resume from checkpoint directory.
                  See Mace4-Checkpoint-Resume.txt.

TPTP flags:

  -tptp           Force TPTP input mode.
  -ladr_out       Force LADR-format model output with TPTP input.
  -ladr-out       Compatible with interpformat, isofilter, and other
                  LADR model tools. See Mace4-TPTP-HOWTO.txt.

Other:

  -c              Compatibility mode. Ignore unrecognized set/clear/
                  assign commands in input. Useful for running Mace4
                  on input files written for Prover9.
  -e <n>          Progress reports to stderr every n seconds during
                  search. Maps to: assign(report_stderr, <n>).
  help            Print usage message and exit.

Positional arguments:

  problem.p       Read TPTP input from file. Auto-detected by .p
                  extension.
  axioms.ax       Read TPTP input from file. Auto-detected by .ax
                  extension.

TPTP auto-detection:

  TPTP mode is auto-enabled when any of:
    - A .p or .ax file is given (via -f or as positional argument).
    - The -tptp flag is present.
  When TPTP mode is active, models are printed in TPTP finite
  interpretation format and SZS status lines are produced on exit.


Examples:

  mace4 -f problem.p                   TPTP input, default sizes.
  mace4 -n 2 -N 10 -t 60 -f prob.p    Sizes 2-10, 60s timeout.
  mace4 -n 6 -m 1 -f group.in         LADR input, size 6, 1 model.
  mace4 -t 300 -b 4096 -f big.p       5 min, 4 GB memory limit.
  mace4 -c -f prover9_input.in        Compat mode, ignore P9 opts.
  mace4 < input.in                     LADR from stdin.


=============================================
Prooftrans
=============================================

Usage:

  prooftrans [format] [transforms] [-f <file>]

Prooftrans reads Prover9 output (from stdin or -f) and transforms
or reformats the proofs. Input must be native LADR format (use
-ladr_out when running Prover9 on TPTP files).

Output formats (at most one):

  (none)          Default: print proofs as-is (with optional transforms).
  parents_only    Condensed proof: show only clause and parent IDs.
  xml             XML format.
  ivy             Ivy proof object format (for Ivy/ACL2 checker).
  hints           Extract hints for reuse in subsequent searches.
  tagged          Tagged proof format (Bob Veroff format).
  tstp            TSTP format (TPTP standard for proof exchange).

Transforms (can be combined, order matters):

  expand          Expand abbreviated inference steps into primitive
                  steps (instantiate, resolve, paramod, flip).
  renumber        Renumber clause IDs sequentially from 1.
  striplabels     Remove label attributes from clauses.

Other options:

  -f <file>       Read input from file instead of stdin.
  -label <label>  Add label prefix to hints (with hints format only).
  help            Print usage message and exit.

The order of expand and renumber matters: "expand renumber" first
expands then renumbers, while "renumber expand" first renumbers
then expands.


Examples:

  prooftrans < proof.out                     Print proofs as-is.
  prooftrans ivy < proof.out                 Ivy proof object.
  prooftrans hints < proof.out               Extract hints.
  prooftrans parents_only < proof.out        Condensed proof.
  prooftrans expand renumber < proof.out     Expand + renumber.
  prooftrans tstp < proof.out                Convert to TSTP.
  prooftrans hints -label round2 < p.out     Hints with label.
  prooftrans ivy -f proof.out                Ivy, from file.


=============================================
Other Programs
=============================================

All other programs in bin/ accept "help" as a command-line argument
to print a usage synopsis:

    bin/clausefilter help
    bin/isofilter help
    bin/interpformat help

Commonly used programs:

  clausefilter <interps> <test> [options] < clauses
      Filter clauses by evaluation in interpretations.
      Tests: true_in_all, true_in_some, false_in_all, false_in_some.

  interpfilter <clauses> <test> < interps
      Filter interpretations by clause evaluation.
      Tests: all_true, some_true, all_false, some_false.

  interpformat [format] [-f <file>] < interps
      Reformat interpretations.
      Formats: standard, standard2, portable, tabular, raw, cooked,
      tex, xml.

  isofilter [options] < interps
      Remove isomorphic interpretations.
      Options: ignore_constants, check '<ops>', output '<ops>',
      wrap.

  get_interps < mace4_output
      Extract interpretations from Mace4 output.

  tptp_to_ladr < problem.p
      Convert TPTP format to LADR format.

  ladr_to_tptp [-p] [-q] < input.in
      Convert LADR format to TPTP format.
      -p: fully parenthesize.  -q: single-quote TPTP symbols.

  gvizify < proof.out > proof.dot
      Convert proof to Graphviz dot format for visualization.
